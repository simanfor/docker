[SIMANFOR](https://www.simanfor.es) is a Decision Support System that enables the simulation of forest management alternatives.

In this repository you will find public documentation on how to run and use the SIMANFOR simulator on your local machine using **Docker** containers, without worrying about setting up complex dependencies or development environments.

## 🚀 Requirements and Installation

To use SIMANFOR this way, you only need to have Docker installed and running on your operating system:

1. Download and install **Docker Desktop** (for Windows or macOS) or **Docker Engine** (for GNU/Linux systems) from the [official website](https://www.docker.com/products/docker-desktop/).
2. Verify that the Docker service is running by executing in your terminal:
   ```bash
   docker --version
   ```

If you are new to Docker, we recommend opening the interactive `cheatsheet_docker_basic.html` file in any web browser for a quick reference guide.

## 📥 Retrieve the Simulator Image

The official SIMANFOR simulator image is hosted publicly in the GitHub Container Registry. You can view the full list of available images and versions in the public **[🔗 SIMANFOR-Dask Packages](https://github.com/orgs/simanfor-dask/packages)** link, with the complete default image being the one highlighted in the following download example:

```bash
docker pull ghcr.io/simanfor-dask/simulator:latest
```

After the download finishes, verify that the image is saved locally:

```bash
docker image ls
```

### Initial functionality check

To ensure the container runs properly on your machine without errors, you can launch it directly without passing any parameters:

```bash
docker run ghcr.io/simanfor-dask/simulator:latest
```

> [!NOTE]
> This quick check runs a default simulation preconfigured inside the container. However, since containers are isolated and ephemeral environments, any output files generated by this check will be deleted automatically once execution ends. To work with your own data and store the results on your machine, follow the instructions below regarding **local volumes**.

## 💾 Running with Your Own Data (Local Volumes)

To pass your own inventory files (.xlsx) and scenarios (.json) to the simulator, and have the simulator write the output files (.xlsx) directly onto your local machine, you must use **Docker volumes** (`-v`).

Volumes act as shared folders between your local computer (the host system) and the Docker container.

### 1. Recommended folder structure

To prevent errors and simplify command usage, we recommend creating a specific workspace directory on your machine (for instance, named `simanfor_docker/`) that contains exactly this subfolder structure:

```text
simanfor_docker/
├── inputs/            <-- Place your scenario .json file and inventory .xlsx template here
└── outputs/           <-- The simulator will automatically save the .xlsx output files here
```

### 2. Internal scenario configuration (.json)

For the simulator running inside the container to find input files and write output files through the mounted directories, you must properly configure the path variables in your scenario `.json` file:

* **Input path (`input`)**: Must point to `../inputs/[inventory_filename].xlsx`
* **Output path (`output_path`)**: Must point to `../tests/outputs/[desired_prefix]__`

This is because the SIMANFOR simulator runs internally from the `/app/src/` working directory inside the container. Therefore, files in the mounted volumes are accessible relatively by going up one directory level (`../`).

An example of the configuration snippet inside the `.json` scenario file:
```json
{
  "user": "Your Name",
  "overwrite_output_file": "YES",
  "output_path": "../tests/outputs/simulation_results__",
  ...
  "operations": [
    {
      ...
      "variables": {
        "input": "../inputs/masas_puras_Espana-plantilla.xlsx"
      }
    }
  ]
}
```

### 3. Simulation commands by Operating System

Once you have your local folder `simanfor_docker/` set up with your files placed under `inputs/`, open a terminal in that workspace and execute the appropriate Docker command for your system and shell:

#### 🐧 On GNU/Linux and macOS (Bash / Zsh)
We use the `${PWD}` environment variable to automatically fetch the absolute path of the current directory:

```bash
docker run \
  -v ${PWD}/outputs:/app/tests/outputs \
  -v ${PWD}/inputs:/app/inputs:ro \
  ghcr.io/simanfor-dask/simulator:latest \
  -s /app/inputs/test_pure_models_spain_plantilla.json
```

#### 🪟 On Windows (PowerShell)
We use the `${PWD}` variable along with the backtick (`` ` ``) line continuation character:

```powershell
docker run `
  -v ${PWD}/outputs:/app/tests/outputs `
  -v ${PWD}/inputs:/app/inputs:ro `
  ghcr.io/simanfor-dask/simulator:latest `
  -s /app/inputs/test_pure_models_spain_plantilla.json
```

#### 🪟 On Windows (CMD - Command Prompt)
We use the `%cd%` variable along with the caret (`^`) line continuation character:

```cmd
docker run ^
  -v %cd%/outputs:/app/tests/outputs ^
  -v %cd%/inputs:/app/inputs:ro ^
  ghcr.io/simanfor-dask/simulator:latest ^
  -s /app/inputs/test_pure_models_spain_plantilla.json
```

## 📊 Quick-Start Examples

To help you check everything works instantly, we have included two preconfigured test scenarios inside the `ejemplos/` folder:

### Pure forest stands simulation test
1. Open your terminal in the `ejemplos/ejemplo_masas_puras/` folder.
2. Run the appropriate command for your OS, specifying the scenario file:
   * On Linux/macOS:
     ```bash
     docker run -v ${PWD}/outputs:/app/tests/outputs -v ${PWD}/inputs:/app/inputs:ro ghcr.io/simanfor-dask/simulator:latest -s /app/inputs/test_pure_models_spain_plantilla.json
     ```
3. Verify that the simulation completes successfully and that 3 Excel spreadsheets representing the inventory plots projected over time appear in your local `ejemplos/ejemplo_masas_puras/outputs/` directory.

### Mixed forest stands simulation test
1. Open your terminal in the `ejemplos/ejemplo_masas_mixtas/` folder.
2. Run the command for your OS:
   * On Linux/macOS:
     ```bash
     docker run -v ${PWD}/outputs:/app/tests/outputs -v ${PWD}/inputs:/app/inputs:ro ghcr.io/simanfor-dask/simulator:latest -s /app/inputs/test_mixed_models_spain_plantilla.json
     ```
3. Check the results generated in that example's local `outputs/` folder.
